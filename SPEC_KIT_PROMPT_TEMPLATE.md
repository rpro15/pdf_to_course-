# Spec Kit Prompt Template (PDF → Course)

Use this prompt as input for `specify`.

## Command Example

```bash
specify "Создай спецификацию MVP для приложения PDF-to-Course.

Контекст продукта:
- Пользователь загружает PDF-книгу.
- Система разбивает текст по главам.
- Одна или несколько AI-моделей создают сжатое резюме каждой главы, сохраняя общий смысл.
- Цель сжатия по умолчанию: из ~10-15 страниц исходного текста сделать ~1 страницу резюме.
- Каждая глава становится отдельным модулем курса.
- Для каждого модуля нужно сгенерировать 8-10 вопросов по смыслу главы.
- Пользователь может вручную редактировать резюме и вопросы.
- Интерфейс MVP минимальный: загрузка PDF, запуск обработки, просмотр модулей, редактирование, экспорт/сохранение.
- Архитектура должна быть расширяемой: чтобы позже можно было добавлять/заменять модели, дообучать их и подключать другие форматы (EPUB/DOCX/TXT), приоритет пока PDF.

Сформируй:
1) user stories с приоритетами P1/P2/P3;
2) edge cases;
3) функциональные требования FR-xxx;
4) ключевые сущности данных;
5) измеримые success criteria;
6) ограничения MVP (без лишних UI-фич)."
```

## Short Prompt (Fast Start)

```text
Создай spec для MVP "PDF-to-Course":
- Вход: PDF-книга
- Выход: по каждой главе сжатое резюме (примерно 10-15 страниц -> 1 страница) + 8-10 вопросов
- Главы = отдельные модули курса
- Сохранение смысла обязательно
- Человек может редактировать результат
- Минимальный UI
- Архитектура расширяемая под новые модели и новые форматы позже

Верни: User Stories (P1/P2/P3), FR, Edge Cases, Entities, Success Criteria.
```

## Optional Add-ons for Better Specs

Append one or more blocks below to your prompt:

### Quality Constraints

```text
Добавь quality gates:
- Проверка сохранения смысла (manual review rubric)
- Проверка покрытия ключевых понятий главы
- Проверка качества вопросов (факт/понимание/применение)
- Логи model/version/prompt для воспроизводимости
```

### Technical Constraints

```text
Технические ограничения:
- Обработка больших PDF чанками
- Возобновляемая обработка после сбоя
- Очередь задач для долгих запусков
- Хранение исходного извлеченного текста и финального резюме
```

### Roadmap Constraints

```text
Добавь roadmap после MVP:
- Поддержка EPUB/DOCX/TXT через адаптеры
- A/B сравнение нескольких моделей
- Возможность дообучения/подключения улучшенных моделей без изменения UI
```
